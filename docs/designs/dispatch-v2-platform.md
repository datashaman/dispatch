## Vision

### 10x Check
Transform Dispatch from "a webhook server one developer runs locally" to "the operating system for AI-powered software development." Any developer installs the GitHub App, picks repos, and is running in 60 seconds. Agents have persistent memory across repos. Rules are composable and shareable via templates. Cost control is first-class. Agent quality is measurable and improves over time. Multi-agent pipelines chain analysis → implementation → review.

### Platonic Ideal
A clean dashboard showing repos, activity, and monthly cost. One-click repo enablement. Template library for instant rule setup. Real-time streaming of agent reasoning. End-of-month reports: "47 issues triaged, 12 PRs created, 8 merged without edits, $23.47 spent." The feeling: "I have an infinitely patient, always-available junior developer who gets better every week."

## Implementation Order

1. **gh CLI → GitHub API** (foundation — unblocks repo picker and Docker)
2. **GitHub App repo picker** (unblocks onboarding flow)
3. **Live streaming via Reverb** (independent, high UX impact)
4. **Cost dashboard & budgets** (independent)
5. **Rule templates** (independent, needs good UX foundation)
6. **Agent run diff view** (small, independent)
7. **Agent quality feedback** (needs agent runs to rate)
8. **Multi-agent pipelines** (most complex, last)
9. **Prompt injection defense** (can be done at any point)
10. **Docker Compose** (after gh CLI migration proves containerizable)

## Design Specifics — Anti-Slop

Things that must NOT look generic:

- **Diff view:** GitHub-style unified diff. Red bg for removed lines, green bg for added. File path headers with `a/` `b/` prefix. Monospace font. Not a fancy split-pane editor — keep it simple and familiar to developers who read diffs daily.
- **Live streaming steps:** Not a chat-style message list. Each step is a collapsible block: header shows tool name + duration, body shows input/output. Tool calls get a subtle blue-tinted left border. Tool results get green-tinted. This matches the existing pattern on the webhook detail page.
- **Cost chart:** Not a generic Chart.js demo. Single color (the project's accent, e.g., indigo-500 in dark mode). No gradient fills. No legend if only one series. Minimal axes. The number matters, not the chart chrome.
- **Template cards:** Not Bootstrap cards with icons. Each card shows the rule's narrative summary ("When issues.opened → triage and label") in the same format as the rules page. The familiar pattern signals "this is a real rule, not marketing."
- **Budget alerts:** Not red banners. Amber background (matching the existing prompt preview tint on the rules page). Conversational tone: "repo-beta has used $18.40 of its $20 budget this month" not "WARNING: Budget threshold exceeded."

## Design System Notes (see DESIGN.md)

**Existing patterns to reuse:**
- `flux:sidebar.item` with icon + label for nav entries → Cost and Templates get sidebar items
- `flux:badge` with color variants for status → reuse for budget status (green=ok, amber=warning, red=exceeded)
- `flux:modal` for detail views → agent run detail already uses this, diff + quality tabs go inside
- `flux:table` with `flux:table.row` → cost breakdown table, template list
- 4-column stat grid (zinc-800 cards, mono numbers) → cost dashboard top stats
- Expand-for-detail chevron toggle → template preview (click to see full prompt)
- Amber-tinted `pre` block → prompt preview in templates (same as rules page)
- `wire:poll.5s` → replaced by Reverb Echo listener for live streaming
- Toast notifications via Flux → all success/error confirmations

**New components needed:**
- Toggle switch for repo picker (Flux has `flux:switch`)
- Inline editable number for budget setting (Flux `flux:input` with edit mode)
- Tab component for agent run modal (Flux `flux:tab` / `flux:tabs`)
- Bar chart for cost (lightweight JS, Alpine-driven, or a minimal chart lib)
- Diff renderer (server-rendered HTML with Tailwind classes, no heavy JS lib)

### Accessibility
- **Keyboard navigation:** All toggles, buttons, and tabs reachable via Tab key. Repo toggles operable with Space. Modal closeable with Escape. Tab switching with arrow keys.
- **Screen readers:** Agent run steps use `role="list"` with `aria-live="polite"` for streaming updates. Status badges have `aria-label` (not just color). Budget alerts use `role="alert"`.
- **Touch targets:** Minimum 44x44px for all interactive elements. Repo toggle switches, quality feedback buttons, template install buttons.
- **Color contrast:** All text meets WCAG AA (4.5:1 for normal, 3:1 for large). Status colors (green/amber/red) always paired with text labels or icons — never color-only.
- **Motion:** Live streaming step animations respect `prefers-reduced-motion`. If reduced, steps appear instantly without slide-in.

## Key Technical Decisions

- **Real-time:** Laravel Reverb (WebSocket) over raw SSE — native Laravel, better connection management
- **GitHub operations:** HTTP API with installation tokens, replacing gh CLI entirely
- **Prompt security:** Structural separation (XML tags for untrusted content) + bash sandbox hardening
- **Pipelines:** `depends_on` field in dispatch.yml rules, topological sort, sequential dispatch with output passing
- **Cost control:** Per-project budgets with configurable action (pause/warn/continue), atomic budget checks via DB transactions
- **Cost chart:** Alpine.js + CSS bars (styled divs with Alpine tooltips). Zero JS dependencies. No Chart.js.
- **Diff view:** Server-side rendered HTML. PHP generates pre-formatted diff with Tailwind +/- coloring. No client-side JS library.
- **Rule templates:** PHP config files in `config/templates/`. Versioned in git, no DB table, no seeder. Install reads config and writes to dispatch.yml.
- **Quality feedback:** Tied to authenticated user (`user_id` on feedback record). Binary thumbs + optional text.
- **Budget alerts:** In-app only for V2 (banner on /cost page). Email/Slack notifications deferred to V3.
